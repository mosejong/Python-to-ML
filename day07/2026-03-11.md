## 오늘 배운 내용 요약

### Git 사용법

#### 1. 터미널에서 자주 사용하는 명령어
- `pwd` : 현재 위치 확인
- `cd` : 경로 이동 (`..` 사용 시 상위 폴더로 이동)
- `mkdir` : 폴더 생성
- `ls` : 파일 목록 확인
- `ls -al` : 숨김 파일까지 포함하여 확인

---

#### 2. Git을 사용하는 이유
로컬 저장소와 원격 저장소(GitHub)를 연결하여 코드의 변경 이력을 관리하기 위해 사용한다.

---

#### 3. 파일을 GitHub 등 저장소에 올리는 과정

1️⃣ **Working Directory**  
- 새로운 파일이 생성된 상태  
- VS Code에서 파일 옆에 `U (Untracked)` 표시가 나타난다.

2️⃣ **Staging Area (준비 영역)**  
- `git add .` 실행 시 파일 옆에 `A (Added)` 표시가 나타난다.  
- commit을 하기 전 준비 단계

3️⃣ **Git Repository**  
- `git commit -m "message"` 실행 시 저장소에 기록된다.  
- 하나의 버전이 생성된 상태

---

#### 4. 주요 Git 명령어  
( `.` 대신 파일명을 작성하면 개별 파일만 적용 )

- `git add .`  
  → 모든 파일을 Staging Area로 보낸다

- `git reset .`  
  → Staging Area에 올라간 파일을 다시 Working Directory 상태로 되돌린다

- `git status`  
  → 현재 Git 상태를 확인한다

- `git commit -m "message"`  
  → 변경 사항을 기록하며 버전을 생성한다

- `git commit --amend`  
  → 직전에 만든 commit을 수정할 때 사용  
  → vim 화면에서 수정 후 `:wq` 로 저장 후 종료

- `git push`  
  → 로컬 저장소의 내용을 원격 저장소(GitHub)로 업로드

- `git pull`  
  → 원격 저장소의 내용을 로컬 저장소로 가져온다

- `git branch`  
  → 협업 시 작업을 분리하기 위한 브랜치 생성 및 관리  
  → 삭제 : `git branch -d 브랜치이름`
