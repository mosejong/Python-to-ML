# 공공데이터 API 실습 - 함평군 버스 위치 데이터 수집 및 시각화

## 1. 실습 목표
공공데이터포털 OpenAPI를 활용해 버스 실시간 위치 데이터를 수집하고,  
이를 pandas DataFrame으로 정리한 뒤 지도 시각화까지 진행해보았다.

---

## 2. 진행 내용

### 2-1. API 호출
- 공공데이터포털 OpenAPI를 사용해 버스 위치 정보를 요청
- `requests.get()`으로 응답을 받고 JSON 형태로 파싱
- 지역 코드(`stdgCd`)를 이용해 특정 지역 데이터를 조회

### 2-2. 데이터 확인
- 처음에는 일부 지역(서울 등)이 조회되지 않아 `NODATA_ERROR`가 발생
- 이후 **전라남도 함평군 (`4686000000`)** 데이터는 정상적으로 조회되는 것을 확인
- 이를 통해 API 사용 시
  - 서비스키 상태
  - 지역 코드
  - 실제 적재된 데이터 존재 여부
  를 함께 확인해야 한다는 점을 알게 되었다

### 2-3. 페이지네이션 처리
- 응답 데이터의 `totalCount`, `pageNo`, `numOfRows` 값을 확인
- 반복문을 통해 여러 페이지를 순회하며 전체 데이터를 수집
- 최종적으로 **총 800건의 함평군 버스 위치 데이터**를 수집하였다

### 2-4. DataFrame 변환
- 수집한 JSON 데이터를 `pandas.DataFrame`으로 변환
- 주요 컬럼:
  - `stdgCd` : 지역 코드
  - `lclgvNm` : 지역명
  - `rteNo` : 노선번호
  - `vhclNo` : 차량번호
  - `lat`, `lot` : 위도/경도
  - `oprSpd` : 운행 속도
  - `totDt` : 수집 시각

### 2-5. 지도 시각화
- `folium` 라이브러리를 사용해 버스 위치를 지도 위에 표시
- 노선별로 색을 다르게 지정해 시각화 진행

---

## 3. 실습 중 발생한 문제

### 3-1. 서울/일부 지역 조회 실패
- 서울, 남양주 등 일부 지역은 `NODATA_ERROR`가 발생
- 반면 함평군은 정상 조회
- 코드 오류라기보다 **API에 실제 적재된 데이터 범위 문제일 가능성**을 확인했다

### 3-2. 지도에 이상한 위치가 표시됨
- 지도 시각화 시 일부 점이 한국이 아닌 전혀 다른 위치에 표시되었다
- 확인 결과 일부 데이터의 좌표값이 `(0.0, 0.0)` 으로 들어가 있었다
- 이 값은 실제 위치가 아니라 **좌표 누락 또는 이상치 데이터**로 판단했다

### 3-3. 이상치 제거
- `(0,0)` 좌표를 제거한 뒤 다시 시각화
- 정상적인 버스 위치만 지도에 표시되도록 수정했다

---

## 4. 배운 점
- OpenAPI 실습에서는 단순히 요청만 보내는 것이 아니라
  - 인증키 상태
  - 지역 코드
  - 응답 구조
  - 실제 데이터 존재 여부
  를 함께 확인해야 한다
- 페이지네이션을 처리하면 한 페이지가 아니라 전체 데이터를 수집할 수 있다
- 시각화는 단순히 보기 좋게 만드는 용도뿐 아니라, **이상치 탐지에도 매우 유용하다**
- 특히 `(0,0)` 좌표처럼 표에서 놓칠 수 있는 문제를 지도에서 쉽게 발견할 수 있었다

---

## 5. 느낀 점
이번 실습을 통해 API 호출 → JSON 파싱 → DataFrame 변환 → 지도 시각화까지  
데이터 처리의 한 흐름을 직접 경험할 수 있었다.

특히 지도 시각화 과정에서 이상치를 직접 발견하고 수정한 점이 인상 깊었다.  
단순히 데이터를 가져오는 것에서 끝나는 것이 아니라,  
**정제와 검증 과정도 중요하다**는 점을 체감했다.
