## 📅 2026-03-18

### 🧠 Today I Learned

#### 1. 실제 데이터 전처리 과정
- 데이터 분석에서 전처리 과정은 전체 작업의 약 80%를 차지할 만큼 핵심적인 단계이며, 데이터의 품질이 결과의 정확도를 좌우함
- 주요 컬럼을 먼저 확인하여 데이터 구조를 파악하는 것이 중요함

#### 2. 프로젝트 1 - 공연 및 축제 데이터 분석
- 문자열로 되어있는 날짜/시간 데이터를 datetime 타입으로 변환하여 활용
- 입장연령, 요금 등 다양한 형태의 데이터를 일관된 형태로 정제
- 조건 필터링을 통해 기간, 금액대, 키워드 기반 분석 수행
- 시각화를 통해 지도, 그래프, 워드클라우드 등 다양한 방식으로 데이터 표현 가능

#### 3. 프로젝트 2 - 상권 및 매출 데이터 분석
- 데이터 로드 후 주요 컬럼 확인을 통해 구조 파악
- 불필요한 컬럼 제거 및 데이터 정리
- 업종, 시간, 성별 등 다양한 기준으로 필터링 및 분석 수행
- groupby를 활용하여 데이터 집계 (sum, mean)
- reset_index를 활용하여 결과 데이터를 재사용 가능한 형태로 변환

#### 4. 데이터 구조 이해의 중요성
- 서로 다른 데이터셋을 결합하기 위해서는 공통된 기준(키)이 필요함
- 상권 데이터와 매출 데이터는 기준 단위가 달라 단순 merge가 불가능함
- 데이터 분석에서는 코드보다 데이터 구조를 이해하는 것이 더 중요함

---

### 💡 Insight

- 엑셀에서 수동으로 처리하던 작업을 코드로 자동화할 수 있어 효율성이 크게 향상됨
- sum(총합)과 mean(평균)의 차이에 따라 분석 결과 해석이 달라짐
- 데이터 분석은 단순히 결과를 보는 것이 아니라, 왜 그런 결과가 나왔는지 해석하는 과정이 중요함

---

### ⚠️ Trouble Shooting

- 파일 경로 문제로 데이터 로드 실패 → 작업 디렉토리 확인으로 해결
- 한글 파일명 문제 → os.listdir()로 실제 파일명 확인
- 데이터셋 간 merge 시 공통 컬럼이 없어 결합 불가 문제 발생
