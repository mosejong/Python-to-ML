## 오늘 배운 내용 요약

### 1. 이미지 데이터의 기본 개념
- CNN은 대표적인 딥러닝 모델로, 이미지 분석 작업에서 강력한 성능을 보인다.
- CNN이 이미지 데이터를 잘 학습하는 원리를 이해하려면 먼저 이미지 데이터 자체를 이해할 필요가 있다.
- 이미지 파일 형식으로는 `JPG`, `PNG` 등이 대표적이다.

### 2. 픽셀(Pixel)
- 이미지는 픽셀이라 불리는 작은 점들의 집합으로 구성된다.
- 각 픽셀은 색깔 또는 밝기 정보를 가진다.
- 이러한 픽셀이 가로와 세로 방향으로 배열되어 2차원 이미지를 형성한다.

### 3. 컬러 이미지와 흑백 이미지
#### 컬러 이미지
- 각 픽셀은 3개의 값을 가진다.
- 이 3개 값은 `RGB 채널`을 의미한다.
- 각 채널은 8비트 값으로 표현된다.
- 총 24비트(`8비트 x 3채널`) 정보를 가진다.
- 이론적으로 `2의 24승`개의 색을 표현할 수 있다.
- 각 채널 값이 클수록 해당 색의 세기가 강해진다.

#### 흑백 이미지
- 각 픽셀은 1개의 값만 가진다.
- 이 값은 밝기 정보를 의미한다.
- 마찬가지로 8비트 값을 가지며, 값이 클수록 흰색에 가깝다.

## PyTorch 이미지 분류 파이프라인 구현

### 1. 데이터 준비
- `train.csv`, `test.csv`를 불러와 이미지 경로와 라벨을 확인했다.
- `LabelEncoder`를 사용해 문자열 라벨을 숫자 라벨(`label_idx`)로 변환했다.
- `train_test_split()`으로 학습용 데이터와 검증용 데이터를 분리했다.
- train/test 이미지 크기를 확인한 결과 모두 `(64, 64)`였다.
- 따라서 업스케일링 없이 원본 이미지 기준으로 실습을 진행했다.

### 2. Dataset / DataLoader
- `Dataset` 클래스를 직접 구현해 이미지 경로를 실제 이미지 텐서로 변환했다.
- `Image.open(...).convert("RGB")`로 이미지를 불러왔다.
- `ToTensor()`를 적용해 텐서로 변환했다.
- `ToTensor()`를 사용하면 픽셀 값 `0~255`가 `0~1` 범위로 자동 변환된다는 점을 배웠다.
- `DataLoader`를 사용해 배치 단위로 데이터를 불러왔다.

### 3. 텐서 shape 이해
- PyTorch 이미지 텐서는 `(채널, 높이, 너비)` 형태를 사용한다.
- 예: `torch.Size([3, 64, 64])`
- 여기서 `3`은 RGB 채널, `64 x 64`는 이미지 크기이다.
- 배치로 묶이면 `(batch_size, channel, height, width)` 형태가 된다.
- 예: `torch.Size([32, 3, 64, 64])`

### 4. CNN 모델 구현
- 처음에는 Keras 방식과 헷갈렸지만, 현재 데이터 로드는 PyTorch 방식이므로 PyTorch로 통일했다.
- `torch.nn as nn`을 사용해 CNN 모델 클래스를 직접 구현했다.
- `__init__()`에서는 Conv, Pool, FC 레이어를 정의했다.
- `forward()`에서는 데이터를 레이어에 통과시키는 흐름을 작성했다.

### 5. feature map 크기 계산
- 입력 이미지 크기는 `64 x 64`
- `padding=1`, `kernel_size=3`인 Conv는 가로세로 크기를 유지한다.
- `MaxPool2d(2,2)`를 거치면 가로세로 크기가 절반으로 줄어든다.
- 따라서 `64 -> 32 -> 16` 흐름으로 줄어든다.
- 마지막 feature map의 shape를 기준으로 `Linear` 입력 크기를 계산해야 한다.
- shape mismatch 에러를 겪으면서 Conv 출력 채널 수가 바뀌면 `fc1` 입력 크기도 함께 수정해야 한다는 점을 이해했다.

### 6. 손실함수와 softmax 관계
- 마지막 출력층 뒤에는 softmax를 직접 넣지 않았다.
- 이유는 `nn.CrossEntropyLoss()`가 내부적으로 softmax 역할까지 포함해서 계산하기 때문이다.
- 따라서 모델 마지막은 `self.fc2(x)` 형태의 logits를 출력하면 된다.
- 예측 클래스를 구할 때는 `torch.argmax(outputs, dim=1)`을 사용했다.

### 7. 학습 루프 직접 구현
PyTorch는 Keras처럼 `model.fit()`을 사용하지 않고, 학습 루프를 직접 작성해야 한다.

학습 순서:
1. `optimizer.zero_grad()`
2. `outputs = model(images)`
3. `loss = criterion(outputs, labels)`
4. `loss.backward()`
5. `optimizer.step()`

### 8. 검증 루프 구현
- `model.eval()`로 평가 모드로 전환했다.
- `with torch.no_grad():`를 사용해 기울기 계산을 비활성화했다.
- `torch.argmax(outputs, dim=1)`로 예측 클래스를 선택했다.
- `(preds == labels).sum().item()`으로 맞춘 개수를 누적했다.
- `correct / total`로 검증 정확도(`Val Acc`)를 계산했다.

### 9. 진행도 표시 및 best model 저장
- `tqdm`을 사용해 epoch 진행도를 표시했다.
- 검증 성능이 좋아질 때마다 `"best model 저장!"`이 출력되도록 구성했다.
- 실습 결과 검증 정확도가 약 `0.45`까지 상승하는 것을 확인했다.

### 10. GPU 환경 이슈
- 로컬 환경에서 `torch.__version__`, `torch.version.cuda`, `torch.cuda.is_available()`를 확인했다.
- 처음에는 CPU 버전 PyTorch가 설치되어 있어 GPU를 사용할 수 없었다.
- 이후 GPU 환경을 새로 만들려고 시도했지만,
- 렌탈 노트북의 애플리케이션 제어 정책 때문에 `torch.dll`이 차단되는 문제를 확인했다.
- 따라서 현재는 CPU 또는 Colab 환경이 더 현실적인 선택지라는 점을 알게 되었다.
